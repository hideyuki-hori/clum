# clum 構文スナップショット（latest）

いま決まっている構文の**現在形だけ**を載せる。経緯・日付・廃止形・未決の議論は載せない（それらは各仕様書が正。この文書と食い違ったら仕様書に従い、この文書を直す）。仕様を変えたら同じコミットでこの文書も更新する。

## ファイルの種類

- `名前.clum` — 通常のモジュール。定義と束縛の入れ物で、既定で密閉（外からは見えない）
- `-.clum` — エントリポイント。`clum build <ディレクトリ>` が実行する唯一のプログラムで、トップレベルに式を並べられる唯一のファイル
- `^.clum` — ディレクトリの公開面。中身は `^名前` の行だけ
- エントリポイント以外のファイルのトップレベルに裸の式は書けない（コンパイルエラー）

## 字句

- インデントはスペース2つで1レベル。タブは全面禁止（テキスト行と文字列リテラルの中身のみ例外）
- コメントは `//` のみ
- 識別子: 定義名は UpperCamelCase（`Card`）、変数・関数名は lower-kebab-case（`save-item`）
- 整数 `42` は `i32`、小数 `3.14` は `f64`。i32 に収まらない整数リテラルはエラー
- 文字列は `'...'` のみ・1行内。`"` の文字列は無い
- 文字列とテキスト行の補間は `{式}`。`{` `}` そのものを書くときは `{{` `}}`
- `.` は2用法: 密着の `page.title` はフィールドアクセス、空白のあとの `.div` はドット前置リテラル

## 記号

- `#` — 宣言
- `@パス` — import。パスは `@` に密着（`@.`、`@./components`）
- `^名前` — export。名前は `^` に密着（`^index`、`^Card`）
- `=` — 束縛。再代入は無い
- `->` — 関数・マッチアームの帰結
- `|>` — パイプ。`x |> f a` は `f a, x`（末尾差し込み）
- `!` — 効果の実行。`Eff<T> -> T`。副作用はここでしか起きない
- `- ` — Vec リテラルの要素マーカー
- `?` — パターンマッチ（後続ステップ）
- 四則演算の記号は無い（`:add` 等の `:` 呼び出しで書く。後続ステップ）

## 宣言と束縛

```
# User name: String, age: i32

# Greet name: String -> String
greet: Greet name -> 'hello, {name}'

x = 1
```

- 束縛はすべて不変。シャドーイングは全面禁止。1定義＝1実装
- 型注釈 `x: T = 式` は任意。`^` した定義（境界）だけ必須

## レコード構築と Vec リテラル

```
pages: Vec<Page> =
  - path: '/index'
    title: 'Home'
  - path: '/about'
    title: 'About'

Recipe
  documents:
    - path: './dist/index.html'
      element: index
  |> build
  |> !
```

- レコードは型名＋1レベル深いフィールド列。`名前: 値` で1行1フィールド、全フィールド必須・宣言と同順
- 構築ブロック内の `|>` 行はフィールド列を終え、構築されたレコード全体を右へ流す
- Vec は `- ` の縦列。要素がレコードなら型名は書かず、最初のフィールドを `- ` に続け、残りを同じ桁に揃える
- リストを書ける位置はフィールド値の下と `=` の右辺の下（どちらも改行して1レベル深く）

## 関数

```
greet 'clum'

items = pages
  |> map page ->
    h .li
      {page.title}
```

- 適用はカンマ区切り。引数は先頭から順に埋まり、arity 未満なら部分適用（残りを待つ関数が返る）
- 無名関数は `引数 -> 本体`（現在は1引数）
- 式の開始行より1レベル深い `|>` 行はパイプの継続

## スコープ

- 値が使えるのは「生まれたスコープ」と「明示的に渡された先」だけ
- ラムダが外側のローカル束縛・引数を暗黙に捕まえるのはコンパイルエラー。必要な値は引数で受ける
- ファイルのトップレベル・import した名前・prelude はどこからでも参照できる
- ループ・一般再帰は無い。反復は `map` などの Vec 演算で書く

## モジュール

```
@.
  index

@./components
  Card

^index
```

- import の対象はディレクトリのみ。`@.` は自分のディレクトリ（兄弟ファイルの公開名。明示 import 必須）、`@./dir` はそのディレクトリの `^.clum` に載っている名前
- `^名前` は独立行・1行1export。同じ名前を2回 `^` するとエラー。改名 export は無い
- `#` 宣言と実装の対は定義名1つ（`^Card`）で公開され、import 側でも両方が見える（双方向随伴）

## 効果

- 効果は `Eff<T>` の値。作っただけでは何も起きず、`!` の位置でだけ実行される
- 同一ブロック内の `!` は記述順に実行される。処理系は並べ替えも除去もしない

## html ターゲット

```
^index

index: Html = h .html
  h .head
    h .meta charset 'utf-8'
    h .title
      clum
  h .body
    h .a href '/about', class 'nav-link'
      これはテキスト
    h .ul
      {items}
    Card 'お知らせ'
      h .p
        本文
```

- 要素は `h .tag`。子は1レベル深いインデントブロック（`->` は書かない）。`h` は値ではなく特別形式
- 属性は `名前 値` をカンマ区切り。真偽属性は名前だけ（`disabled`）。`on-*` は書けない。未知の属性はエラー（`data-*`・`aria-*` は自由）
- 子ブロックの行の意味は行頭だけで決まる: `h` ＝要素、UpperCamelCase ＝コンポーネント、`{` ＝埋め込み、それ以外＝テキスト行
- 裸の式行は書けない。式は `{式}` で埋め込み、複数行の式はトップレベルで束縛してから `{名前}` で埋める
- 連続するテキスト行は何も挟まず連結される（改行・インデントは出力に残らない）
- 行頭が UpperCamelCase や `h ` になるテキストは `{'...'}` で書く
- コンポーネントのタグは定義名（`Card`）。末尾引数が `Html` なら子はちょうど1個、`Vec<Html>` なら任意個
- void 要素（`br`、`img`、`meta` 等）は子を持てない。出力は `<br>` 形式
- テキストの `<` `>` `&`、属性値の `&` `"` `<` は自動エスケープ。raw 出力は無い
- 出力は無整形（改行・インデントを付けない）。ルートが `h .html` なら `<!DOCTYPE html>` を先頭に付与、それ以外は fragment

## エントリポイントとビルド

```
@.
  index

Recipe
  documents:
    - path: './dist/index.html'
      element: index
  |> build
  |> !
```

- `Document` は `path`（`-.clum` のあるディレクトリ基準の出力先）と `element`（出力する `Html` 値）
- html 語彙（`h`・`Html`・`Recipe`・`Document`・`build` 等）と `map`・`true`・`false` は prelude として import 不要

## 決定済みだが後続ステップの構文

- `? 値` ＋ `:is` / `:else` のパターンマッチ、`:any` の直和型
- `:` 呼び出し（`1 :add 2` の中置、述語、制約）
- `Eff` の合成、`@js` / `@rust`、css / js / wasm ターゲット、`:share`

## 実装の追随状況（仕様に binary がまだ届いていない箇所。追随したら行を消す）

- export: 実装はまだ旧 `:pub`（`^名前` は未実装）
- import: 実装はまだファイル単位（`@./index` がファイルを指す）。ディレクトリ import・`^.clum` は未実装
- CLI: 実装はまだ `clum build ./main.clum`（ファイルパス指定）。`-.clum` と `clum build <ディレクトリ>` は未実装
