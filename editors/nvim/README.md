# clum の nvim シンタックスハイライト

自前 lexer を正とする方針（docs/plan/002.md トラック C-2 案 (b)）の第1弾。
行頭トークンの字句クラスに基づく正規表現ハイライトであり、名前解決を要する区別
（子ブロック内のテキスト行とコード行、属性名の検査など）は行わない。
精密化は LSP セマンティックトークン（トラック C-3）で行う。

## インストール

lazy.nvim の場合:

```lua
{ dir = '~/h/clum/editors/nvim' }
```

素の runtimepath に足す場合（init.lua）:

```lua
vim.opt.runtimepath:append('~/h/clum/editors/nvim')
```

## 含まれるもの

- `syntax/clum.vim` — ハイライト定義
- `ftdetect/clum.vim` — `*.clum` の filetype 判定
- `ftplugin/clum.vim` — インデント2スペース・`//` の commentstring

## ハイライトの対応

- `#` 宣言マーク・`@` import・`^` export ブロック・`- ` リストマーカー
- 定義名（UpperCamelCase）・フィールド名／注釈つき束縛名・`:` 語（`:pub` 等）
- `h` 特別形式・ドット前置リテラル（`.div` 等。フィールドアクセスの `.` とは区別）
- 文字列（`{式}` 補間・`{{` `}}` エスケープを含む）・数値・`{式}` 埋め込み
- `|>` `->` `=` `!`・`//` コメント

## 既知の割り切り（粗版）

- 子ブロック内のテキスト行は無色（ブロック文脈の追跡をしない）
- 行頭が UpperCamelCase のテキストは定義名と同色になる（言語仕様上も
  コンポーネント参照と解釈される位置のため、挙動としては仕様に忠実）
- 属性名（`href` 等）は無色
